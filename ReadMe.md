# OmniNodeRu PHP SDK

Легковесная библиотека на PHP для работы с асинхронным API нейросети **OmniNode.Ru**. 

Позволяет отправлять запросы ИИ, отслеживать статус выполнения задач вручную или обрабатывать автоматические уведомления (Webhooks) с валидацией безопасности (HMAC-SHA256).

## Установка

Просто скопируйте класс `OmniNodeRu` в ваш проект и подключите его:
```php
require_once 'path/to/OmniNodeRu.php';
```

# Быстрый старт

Для начала работы вам понадобятся два токена, которые можно получить в личном кабинете OmniNode:
1. `InToken` (Входящий токен) — для отправки запросов.
2. `OutToken` (Исходящий токен) — для проверки подлинности вебхуков.

```php
$inToken = "ВАШ_IN_TOKEN";
$outToken = "ВАШ_OUT_TOKEN";

$omni = new OmniNodeRu($inToken, $outToken);
```

# Использование

1. Отправка запроса к нейросети (Ask)

Так как API работает асинхронно, метод Ask возвращает уникальный ID запроса (request_id). Сам ответ нейросети будет готов позже.
```php
// Отправка простого запроса
$requestId = $omni->Ask("Привет! Расскажи анекдот про программистов.");

if ($requestId) {
  echo "Запрос успешно принят. ID задачи: " . $requestId;
} else {
  echo "Ошибка при отправке запроса.";
}
```

Использование контекста (истории диалога)
Вы можете передавать историю переписки в формате массива:

```php
$history = [
    ["role" => "user", "content" => "Привет, меня зовут Никита."],
    ["role" => "assistant", "content" => "Привет, Никита! Рад знакомству. Чем могу помочь?"]
];

$requestId = $omni->Ask("Как меня зовут?", $history);
```

---

2. Получение результата

Получить ответ от нейросети можно двумя способами: через Вебхуки (рекомендуется) или Ручной опрос (Polling).

Способ А: Использование Вебхуков (Автоматически)
Настройте в кабинете OmniNode адрес вашего скрипта-обработчика (например, https://yourdomain.com/webhook.php).

Внутри webhook.php вызовите метод Handler(). Он сам проверит подпись безопасности от OmniNode и передаст чистые данные в вашу callback-функцию:
```php
// webhook.php
require_once 'OmniNodeRu.php';

$omni = new OmniNodeRu("ВАШ_IN_TOKEN", "ВАШ_OUT_TOKEN");

$omni->Handler(function($requestId, $result) {
  // Этот код выполнится только если запрос подлинный и подпись верна
  
  // Пример обработки: сохранение в файл или базу данных
  $logMessage = "Запрос ID: $requestId успешно выполнен.\nОтвет: " . print_r($result, true) . "\n";
  file_put_contents('results.log', $logMessage, FILE_APPEND);
  
  // Важно: библиотека автоматически отправит HTTP 200 OK обратно серверу OmniNode
});
```

Способ Б: Ручной опрос статуса (Polling)
Если вы не можете принимать вебхуки, вы можете периодически опрашивать статус выполнения задачи.
```php
$requestId = "ID_ПОЛУЧЕННЫЙ_РАНЕЕ";

$status = $omni->Check($requestId);

echo "Текущий статус: " . $status["status"]."<hr>Ответ: ".$status["response"]; 
// Возможные статусы зависят от API (например: completed, processing, pending)
```

# Безопасность

Библиотека обеспечивает высокий уровень безопасности при работе с вебхуками:

• Валидация HMAC-SHA256: Каждый входящий вебхук подписывается сервером OmniNode. Метод Handler сверяет эту подпись с использованием вашего секретного $OutToken.

• Защита от Timing Attacks: Сравнение подписей происходит с помощью безопасного метода сравнения строк за константное время (hash_equals), что делает невозможным подбор подписи методом анализа времени ответа сервера.
